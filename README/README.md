PROJECT 3

"SERVERLESS MESSAGING USING SNS, SQS & LAMBDA"


OBJECTIVE:

Implement reliable asynchronous messaging

Decouple producers and consumers

Handle message failures gracefully


PROBLEM STATEMENT:

Direct service-to-service communication causes cascading failures

No buffering during traffic spikes


ARCHITECTURE OVERVIEW:

SNS publishes messages

SQS queues ensure message durability

Lambda processes messages asynchronously


AWS SERVICES USED:

Amazon SNS

Amazon SQS

AWS Lambda

AWS Storage Service


BUSINESS IMPACT:

Improved fault tolerance

Reliable message delivery

Production-grade async processing
